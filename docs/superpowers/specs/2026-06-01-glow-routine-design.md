# Glow Routine - 微信小程序完整体产品企划与设计规格

- **产品名称**：Glow Routine（今日护肤日历 / AI 护肤搭子）
- **产品定位**：面向白领、轻熟龄、成分党与护肤新手的“日常护肤操作系统”，帮助用户把囤货、成分、天气、肤感、作息和消费冲动整理成每天可执行的护肤计划。
- **核心承诺**：少纠结、少乱买、少叠猛药、少浪费，坚持一套更理智的日常护理节奏。
- **设计美学**：极简高级、莫兰迪奶茶暖沙色系（`#F7F4EF`）、轻毛玻璃质感、低饱和功能色。产品气质应像“懂成分但不焦虑的朋友”，而不是医疗问诊工具。

---

## 1. 产品判断与完整体愿景

护肤用户的主要矛盾已经从“买不到产品”转向“拥有太多信息、太多产品、太多种草内容，却缺少稳定的日常决策系统”。传统护肤 App 往往押注 AI 测肤、社区内容或商品推荐，容易陷入低频、同质化和合规风险。Glow Routine 的完整体不做“诊断皮肤”，而是做三个高频场景的连接器：

1. **今天怎么用**：基于用户肤质、护肤品柜、天气、经期/作息、近期肤感，生成晨间/夜间 Routine。
2. **东西怎么管**：管理开封保质期、空瓶计划、成分冲突、功能重复和库存浪费。
3. **想买先冷静**：在购买前识别 hype、重复、冲突和预算压力，给出可保存、可分享的冷静分析。

完整体的核心创意是 **Glow OS（护肤节律系统）**：用户不是每天问 AI“我该怎么办”，而是把自己的产品、肤感和生活节律交给系统，系统每天给出清晰的“可执行护肤日历”。AI 负责解释、总结、识别和生成内容；日常计划由规则引擎和用户数据驱动，保证低成本、可控、合规。

## 3. 完整体功能地图

### 3.1 Glow Calendar：今日护肤日历

**目标**：让用户打开小程序后 10 秒内知道今天晨间/夜间该用什么、什么别用、为什么。

- **今日红绿灯**：综合天气、用户近期泛红/爆痘/干燥打卡、经期敏感标记、前一晚是否使用高刺激活性，输出“稳定 / 观察 / 降刺激”三种状态。
- **晨夜双 Routine**：以步骤卡呈现洁面、水、精华、乳霜、防晒、特殊护理。每张卡绑定用户护肤品柜里的真实产品。
- **智能禁用与降频**：当用户打卡泛红、屏障不稳、连续使用酸/A 醇、睡眠极差时，高刺激产品自动灰显，并给出温和替代。
- **替换与跳过**：用户可横滑替换同类产品，也可跳过某一步。系统记录跳过原因，用于周报分析。
- **解释层**：每个建议有一句简短理由，例如“昨晚用了 A 醇，今晚先保湿修护，让活性间隔一天”。

### 3.2 我的护肤品柜

**目标**：把用户的囤货变成可管理资产，而不是一堆记不清日期和功效的瓶子。

- **货架视图**：按洁面、水、精华、乳霜、猛药、防晒、面膜、身体护理分层。
- **开封保质期 PAO Bar**：绿色表示状态良好，黄色表示建议优先用完，红色表示过期或需要确认。
- **空瓶计划**：系统按过期风险、剩余量、使用频率生成“本月优先消耗清单”。
- **功能重复提醒**：识别多个产品都承担维 C、A 醇、酸类、修护、保湿、防晒等功能，提示“已有同类，不急着买”。
- **成本仪表盘**：统计库存估值、已开封价值、即将过期价值、每次使用成本，强化“少乱买”的核心定位。
- **入库方式**：
  - 免费手动：分类、开封日期、PAO、主活性标签、使用频率。
  - 订阅 OCR：拍包装/成分表，自动提取产品名、分类、PAO、核心成分和使用建议。
  - 截图入库：从购物页或小红书截图识别商品，适合“先加入候选，不立刻购买”。

### 3.3 成分冲突雷达

**目标**：把“成分党知识”产品化成清晰、安全、可执行的提醒。

- **冲突矩阵**：维护酸类、A 醇、过氧化苯甲酰、强效维 C、高浓度烟酰胺、去角质类、修护类等标签之间的搭配关系。
- **频率守卫**：关注连续使用天数和间隔，而不是简单粗暴地永久禁用。
- **个人耐受度**：用户可标记“用后刺痛/泛红/稳定”，系统逐步调低或调高活性频率。
- **护肤日历视图**：用一周时间轴展示活性安排、修护日和防晒提醒，让用户看到节奏，而不是每天临时纠结。

### 3.4 皮肤日记与趋势复盘

**目标**：让用户用 30 秒记录足够有价值的数据，同时避免把产品做成医疗诊断。

- **30 秒打卡**：油腻感、干燥感、泛红、爆痘、刺痛、睡眠、饮食/熬夜/经期/压力触发源。
- **实际使用确认**：从今日 Routine 自动带出产品，用户只需勾掉没用的。
- **Before/After 对比画布**：左右拖拽对比 7 天前、30 天前或自选日期照片；默认不提供美颜滤镜，强调一致光线和同角度提醒。
- **AI 周报/月报**：
  - 输入：日记 JSON、实际使用产品、活性成分、天气和作息标签。
  - 输出：趋势摘要、可能相关因素、下周护肤节奏建议、库存消耗建议。
  - 语气：温和、理智、像朋友复盘，不使用诊断口吻。

### 3.5 小红书避坑冷静分析器

**目标**：在用户“想买”的高冲动瞬间，提供低摩擦的冷静防线。

- **输入方式**：上传种草截图、复制商品文案、输入产品名，或从候选清单选择。
- **分析维度**：
  - 是否与肤质目标匹配。
  - 是否与护肤柜已有产品功能重复。
  - 是否与当前 Routine 中的活性冲突。
  - 是否临近大促冲动购买或预算超标。
  - 卖点是否过度承诺。
- **Routine 模拟器**：买前生成“如果加入这瓶，它会出现在一周哪几天”，让用户直观看到新产品是否真的有位置。
- **冷静拔草卡片**：输出 9:16 海报，包含适合度、重复度、冲突风险、替代方案和一句带幽默感的结论。语气可以犀利，但必须避免羞辱用户或制造容貌焦虑。

### 3.6 社交传播与轻社区

**目标**：不做重社区，不让用户陷入焦虑比较，而是围绕“理智护肤成果”做轻传播。

- **可分享卡片**：空瓶战报、冷静拔草证书、我的本周护肤节律、库存减负成果。
- **匿名模板**：分享默认隐藏价格、照片和敏感肤况，只展示用户主动选择的内容。
- **好友互勉**：可创建 2-5 人护肤坚持小组，只共享打卡天数、空瓶进度和本周状态，不共享面部照片。
- **达人模板拓展**：未来可邀请成分党达人提供“护肤节律模板”，用户导入后仍需经过个人肤质与护肤柜适配。

## 4. 系统架构

### 4.1 架构分层

```text
微信小程序前端
  ├─ 今日日历 / 护肤柜 / 打卡 / 冷静分析 / 分享卡
  ├─ 本地规则预览与缓存
  └─ 图片选择、Canvas 海报、订阅消息

微信云开发
  ├─ 云数据库：用户、产品、打卡、规则、报告、订单
  ├─ 云存储：面部记录照、商品截图、分享卡导出图
  └─ 云函数：
      ├─ routine-engine：生成 Routine 与活性日历
      ├─ ingredient-parser：OCR/LLM 解析产品与成分
      ├─ cooling-analyzer：买前冷静分析
      ├─ report-generator：周报/月报生成
      ├─ moderation-guard：敏感词与合规改写
      └─ billing-entitlement：订阅、广告解锁、权益校验

外部能力
  ├─ OCR / 多模态识别
  ├─ 轻量 LLM 推理与文本生成
  ├─ 天气与城市信息
  └─ 微信支付、激励视频广告、订阅消息
```

### 4.2 AI 编排原则

- **规则先行，AI 补充**：每日 Routine、冲突禁用、频率控制优先由可审计规则引擎完成。
- **AI 不直接给医疗判断**：AI 只做信息提取、解释、复盘和生活护理建议。
- **所有 AI 输出过合规层**：输出前必须经过敏感词扫描、风险话术替换和免责声明拼接。
- **成本可控**：高频动作本地化/规则化；LLM 只在入库、冷静分析、周报/月报等高价值节点触发。

## 6. 关键规则设计

### 6.1 Routine 生成规则

1. 先判断当天状态：稳定、观察、降刺激。
2. 根据晨间/夜间筛选可用产品：状态为 `opened`，未过期，匹配步骤，符合用户偏好。
3. 应用冲突矩阵：酸/A 醇/高强度维 C 等活性不得连续高频叠加。
4. 应用肤感反馈：近 3 天有泛红、刺痛或干燥升高时，降低活性频率。
5. 应用空瓶计划：同等条件下优先推荐临期、已开封较久或剩余量较少的产品。
6. 生成简短解释，并允许用户替换或跳过。

### 6.2 冷静分析评分

- **适配度**：是否符合肤质、目标、预算和当前耐受度。
- **重复度**：是否和护肤柜已有产品功能重叠。
- **冲突度**：是否会挤占当前 Routine 或增加刺激风险。
- **证据稳健度**：卖点是否基于清晰成分与合理浓度区间，避免绝对化承诺。
- **购买时机**：是否已有临期库存、是否正处在大促冲动期、是否本月预算超额。

结论不要只给“买/不买”，而是给出行动路径：先消耗哪瓶、如果买应替换哪瓶、多久后再评估。

## 7. 视觉与交互设计

### 7.1 信息架构

底部 Tab 建议为：

1. **今日**：Routine、红绿灯、天气/肤感提示。
2. **护肤柜**：货架、PAO、空瓶计划、成本仪表盘。
3. **日记**：30 秒打卡、照片对比、趋势报告。
4. **冷静买**：截图分析、候选清单、拔草卡片。
5. **我的**：肤质档案、会员权益、提醒设置、免责声明。

### 7.2 交互原则

- 高频页少文字、强行动：今天用什么、现在打卡、这瓶快过期。
- 专业解释放在二级层：用户想看时再展开成分、规则和原因。
- 分享卡要漂亮但克制：不制造容貌焦虑，不鼓励攀比贵价产品。
- 所有“风险”表达使用生活护理口吻，例如“建议先观察”“今晚降低刺激”“这两类错开更稳”。

## 8. 合规与安全边界

### 8.1 话术边界

- **禁用高风险表达**：诊断、治疗、处方、治愈、临床疗效、皮肤病判断、过敏治疗、消炎祛痘、医学建议。
- **推荐安全表达**：日常护理参考、肤感记录、护肤节奏建议、成分搭配提醒、生活方式复盘。
- **严重情况引导**：当用户输入大面积红肿、持续疼痛、渗出、严重爆发等描述时，停止护肤建议，提示尽快前往正规医院皮肤科。

### 8.2 AI 输出护栏

1. Prompt 层限制 AI 不做诊断和治疗建议。
2. 云函数对输出做敏感词扫描。
3. 命中风险词时自动改写为日常护理建议或转诊提醒。
4. 报告底部固定展示免责声明。
5. 面部照片默认私密存储，不用于公开分享或训练；分享必须由用户主动选择。

## 9. 完整体开发范围与验收标准

完整体可以一次性作为目标范围推进，但工程上仍建议按模块并行开发、按能力逐步合流，最终验收完整产品闭环。

### 9.1 必须完成

- 用户肤质档案、提醒偏好和免责声明确认。
- 护肤品柜的手动入库、OCR/截图入库、PAO、空瓶计划。
- 成分标签、冲突矩阵、频率守卫。
- 今日晨夜 Routine 自动生成、替换、跳过、打卡。
- 皮肤日记、照片存储、Before/After 对比。
- AI 周报/月报、合规过滤、报告解锁。
- 冷静分析器、Routine 模拟器、拔草分享卡。
- 会员权益、广告解锁、订阅校验。
- 基础分享、订阅消息、数据备份与隐私设置。

### 9.2 验收指标

- 用户首次完成肤质档案与 3 个产品入库的流程不超过 3 分钟。
- 打开“今日”页后 10 秒内能理解今日 Routine。
- Routine 生成不依赖 LLM，普通用户每日使用无 token 成本。
- 所有 AI 输出链路都经过合规过滤，不出现医疗诊断式结论。
- 冷静分析必须引用用户护肤柜数据，不能只给泛泛建议。
- 分享卡默认隐藏隐私信息，用户需主动选择公开内容。

---

## 10. 推荐优先级

完整体开发不等于所有页面平均用力。建议优先把 **护肤柜 + Routine + 打卡 + 冷静分析** 做扎实，因为它们构成每日留存和消费防线；社交、达人模板和趋势洞察可以在核心闭环稳定后增强。

最值得保留并强化的创意是：

1. **买前 Routine 模拟器**：比普通成分分析更有差异化，能直接回答“这瓶买了到底有没有位置”。
2. **空瓶计划 + 成本仪表盘**：把护肤管理从变美焦虑拉回理智消费，强化产品人格。
3. **成分冲突雷达 + 护肤日历**：把复杂成分知识变成日历节奏，是 Glow Routine 的长期护城河。
4. **冷静拔草分享卡**：有传播潜力，但必须克制表达，避免攻击用户或诱导焦虑。
