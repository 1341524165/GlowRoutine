## Context

本项目致力于为“白领 / 轻熟龄 / 成分党”女性开发一款微信小程序 “Glow Routine”（今日护肤日历 / AI 护肤搭子）。
目前开发环境为微信原生小程序前端 (`miniprogram/`) 与微信云开发后端 (`cloudfunctions/`)，支持 NoSQL 数据库及云存储。我们需要在这个架构上，实现包含今日 Routine、我的护肤柜、打卡日记及大模型避坑拔草这 4 项新功能在内的 MVP（最小可行性产品），并完成变现模式与合规防御的闭环。

## Goals / Non-Goals

**Goals:**
- **超低启动与运行成本**：日常打卡和今日 Routine 生成为 0 Token 成本（纯前端与云端 JS 规则引擎适配），AI 分析高频请求采用看广告解锁置换。
- **极致的个性化体验**：建立起“护肤柜是单品源，今日 Routine 是作战表，打卡日记是战果积累”的数据闭环联动。
- **高变现潜力与高传播性**：通过小红书卡片式拔草分析证书设计拉动自然裂变传播，个人主体采用激励视频广告秒级套现，企业主体预留订阅会员制。
- **安全顺利过审**：通过严格的话术过滤和免责声明设计，消除微信对“医疗美容/敏感肌诊疗建议”等无资质运营的安全审核红线。

**Non-Goals:**
- **精密自拍测肤**：本 MVP 显式排除通过自拍照片进行精确肤质及病理诊断的功能，避免算法不准导致的信任下降和微信医疗审核封禁。
- **精准商业品牌导购**：MVP 阶段只推荐“成分品类”，不主动收录和推荐具体的商业化妆品品牌，避免合规和高维护成本问题。

## Decisions

### 1. 今日 Routine 使用本地/云端规则引擎生成，取代每日 LLM 实时生成
- **决策理由**：大模型每天实时生成的 Routine 存在步骤随机、格式不稳的弊端，且 API 调用成本极高。使用 JSON 决策树在客户端/云端匹配本地的【护肤柜】单品，不仅 0 成本、反应瞬间，还能 100% 保证活性成分使用顺序的科学性。
- **替代方案**：每日早晚调用一次大模型（API 费用高且极难应对并发）。

### 2. 护肤品柜采用双通道录入（手动胶囊组件 + 订阅/广告解锁 OCR & LLM）
- **决策理由**：白领成分党柜子里的瓶瓶罐罐非常多，手工录入繁琐，拍照 OCR + 大模型结构化成分提取能极大提升高级感。我们将此设计为 9.9 元/月订阅或看 15s 激励广告的会员功能，用高附值变现平衡云函数及 OCR 的 API 成本。
- **替代方案**：完全手动输入（体验单一）或自建百万级化妆品成分商品数据库（维护成本与查询接口调用费用过于庞大）。

### 3. 皮肤趋势分析采取周度聚合 AI 生成，取代每日实时大模型点评
- **决策理由**：皮肤周报可以通过交叉读取用户 7 天的【日记打卡打分 JSON】和这期间的【柜子猛药使用记录（是否高频刷酸/维C防晒）】输出极其深刻的因果关系洞察，比起每日单一数据反馈更加科学，且调用频次由每天降为每周，API 成本直接暴降 85%。
- **替代方案**：每日打卡自动生成 AI 对话（由于数据点薄弱，每天生成的建议容易流于废话且调用成本巨高）。

### 4. 小红书拔草分析选用 LLM 实时知识检索与推理，取代静态化妆品库
- **决策理由**：新晋爆款国货和小众海外护肤品在小红书层出不穷，自建库永远无法追赶流行。直接引入支持联网检索或常识渊博的轻量级大模型（如 Gemini Flash / DeepSeek），通过提取的商品文字实时搜集其主活性成分并进行安全性评估，真正做到 0 库维护成本且“万物皆可查”。
- **替代方案**：纯静态商品成分数据库（开发重、维护烦、遇到新品查不出时用户体验极度受挫）。

### 5. 个人开发者通过“激励视频广告/插屏广告”变现，绕过微信支付主体限制
- **决策理由**：中国微信生态下个人小程序完全无法接入微信支付。我们首创“看 15s 激励视频广告，免费解锁一次 AI 闺蜜避坑分析/AI 趋势周报”的设计，对女性用户来说消费门槛为 0，对开发者来说 eCPM（千次展示收益）在 30~100元左右，完全能轻松覆盖大模型几厘钱的 Token 成本。
- **替代方案**：挂靠企业主体开通微信支付（注册公司成本高、报税繁琐，不适合 MVP 快速验证阶段）。

## Risks / Trade-offs

- **[Risk] 微信审核判定小程序为“无资质行医/敏感肌护肤诊断”导致永久封禁下架** 
  - **Mitigation**：1. UI 界面全局话术脱敏，将“诊断”、“疗效”、“处方”等医学词汇替换为“肤况打卡”、“护肤自评建议”、“避坑小建议”；2. 首次进入强弹窗提示免责声明，明确非医疗建议；3. 云函数增加二次 Output 正则过滤拦截机制，替换大模型可能吐出的医学诊断词。
- **[Risk] 大模型联网 API 调用出现大延迟或连接中断导致用户等待受挫**
  - **Mitigation**：在用户上传截图后，前端通过精美的“AI 闺蜜正在努力看配方表...”的莫兰迪色加载动效消除焦虑；对已查询过的热门商品在云数据库建立缓存库，同商品第二次查询直接秒出缓存结果。
